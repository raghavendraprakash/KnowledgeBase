# Module 2: Requirements Engineering

## Course Outcomes
At the end of this module, the student will be able to:
* **CO1:** Explain the fundamental tasks involved in requirements engineering, including inception, elicitation, elaboration, negotiation, specification, validation, and management.
* **CO2:** Identify project stakeholders and apply appropriate elicitation techniques, such as collaborative gathering and usage scenarios, to define system requirements.
* **CO3:** Develop effective use cases and supplementary UML models (activity and swimlane diagrams) to represent system functionality from a user perspective.
* **CO4:** Perform data and class-based modeling by identifying data objects, attributes, relationships, and analysis classes to build a robust requirements model.
* **CO5:** Evaluate requirements for consistency, ambiguity, and completeness, and perform domain analysis to identify reusable analysis patterns.

## Bloom's Taxonomy-Based Assessment Framework

| Cognitive Level | Focus | Example Task |
| :--- | :--- | :--- |
| **Remember** | Recall facts/definitions | Define the seven tasks of RE. |
| **Understand** | Explain concepts/ideas | Explain the difference between primary and secondary actors. |
| **Apply** | Use information in new situations | Develop context-free questions for a new system. |
| **Analyze** | Draw connections among ideas | Compare collaborative gathering vs. traditional interviews. |
| **Evaluate** | Justify a decision | Critique a Use Case diagram for ambiguities. |
| **Create** | Produce original work | Design a Swimlane diagram for a specific process. |

### Sample Assessment Paper

#### Part A: Foundations (Remember & Understand)
1. List and briefly describe the seven tasks of the requirements engineering process.
2. What is the distinction between functional and non-functional requirements?
3. Define the role of a "Facilitator" in a collaborative requirements gathering session.

#### Part B: Analysis & Application (Apply & Analyze)
4. Analyze a provided Use Case diagram and identify at least two potential stakeholder communication gaps.
5. Scenario: Generate 5 context-free questions to initiate the elicitation process for an automated parking system.
6. Contrast the approach of "Scenario-based Modeling" with "Class-based Modeling."

#### Part C: Evaluation & Synthesis (Evaluate & Create)
7. Evaluate the following requirement for a mobile banking application: "The system must be fast and secure." Discuss why this is inadequate and rewrite it to be measurable and testable.
8. Create a Swimlane diagram representing the workflow of a "Returns and Refunds" process, ensuring clear separation of responsibilities between Customer, Inventory System, and Payment Gateway.
